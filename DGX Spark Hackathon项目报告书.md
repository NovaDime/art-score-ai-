# 基于 DGX Spark 的多模态 AI 智能评估与成长系统
—— DGX Spark Hackathon 项目报告书

## 一、 项目概述
本项目构建了一个基于 DGX Spark 与本地多模态大模型（Qwen2.5-VL-3B-Instruct）的图片作品智能理解与评估系统。系统打通了“自定义标准输入 - 视觉内容解析 - 五维结构化评分 - 可视化反馈”的全链路，将大模型的非结构化自然语言输出转化为可量化、可解释的结构化评估数据，在校园美术与设计类课程场景中形成完整的前后端产品闭环。

## 二、 技术创新性
本项目突破了单纯调用大模型生成宽泛文本的局限，突破传统思路并解决技术痛点，核心创新点在于：

1. **“先理解，后量化”的认知防幻觉机制**：在执行评分前，强制模型优先对图片进行像素级的视觉解析与自然语言描述。通过“看懂图”这一前置步骤锚定上下文，避免了传统模型“盲目套用模板打分”的劣势，极大增强了评分的逻辑自洽与可解释性。
2. **非结构化数据向结构化系统的降维重构**：系统在推理层引入严格的输出约束，将多模态大模型的泛化输出精准拆解为五维评分（Composition, Color, Creativity, Completeness, Theme）、优势/待提升点、综合建议等 JSON 结构化字段。这一设计使得大模型算力能够直接驱动前端雷达图（Chart.js）渲染与数据库（history.json）沉淀。

## 三、 场景落地性
项目贴合真实的物理 AI 行业场景，直击校园美术、设计类作业评估场景的痛点，具备高度的可复制性与商业推广潜力：

1. **教与学双向赋能**：对教师端，通过自定义 Rubric 提供统一评估基线，消除主观偏差并替代高频重复的批阅劳动；对学生端，提供超越单一分数的细粒度雷达图反馈与针对性学习建议，辅助自主迭代。
2. **数据隐私与校园私有化闭环**：传统依赖公有云 API 的方案存在学生作品数据泄露风险。本项目基于纯本地架构，图片数据不出局域网，完美契合高校对敏感数据与隐私保护的合规要求，具备直接向各校园机房或艺术学院工作站推广的条件。

## 四、 技术完整性
本项目功能完整、运行稳定，已构建高内聚、低耦合的全栈产品闭环：

1. **前端展示层**：基于 HTML/JS 与 Chart.js 库，实现无刷新图片预览、评分标准动态下发及五维雷达图的实时渲染。
2. **后端逻辑层**：采用 FastAPI + Uvicorn + Pydantic 构建高并发异步服务，负责路由分发、历史记录落盘（Local JSON）及模型调度。
3. **工程化结构**：项目结构清晰规范，包含完整的环境依赖（`requirements.txt`）、启动/停止脚本（`start_local.sh` / `stop.sh`）及日志管理目录（`logs/`），实现逻辑清晰，可顺利完成演示。

## 五、 平台适配性
充分发挥 DGX Spark 平台的全栈能力，深度压榨硬件底座特性：

1. **显存红利转化**：充分利用 DGX Spark Large unified memory – 128GB 的统一内存架构，系统能够以极低的硬件门槛和零显存溢出风险，常驻运行并处理高分辨率图像的 Qwen2.5-VL-3B-Instruct 多模态推理任务。
2. **后续算力优化空间**：针对后端 FastAPI 接口，未来可无缝引入 DGX Spark 原生支持的 TensorRT-LLM 引擎，通过 In-flight Batching 应对期末/课后大量学生并发上传作品的峰值吞吐压力。

## 六、 演示效果设计
演示流程设计确保流畅、展示清晰、逻辑严谨，直观呈现核心价值：

1. **初始化**：教师视角录入具体的评分 Rubric（例如：“色彩维度需考察是否使用了高饱和度对比色”）。
2. **执行**：上传测试图像（`tests/test.jpg`），展示系统在毫秒至秒级的推理延迟内完成响应。
3. **展示**：终端实时打印模型底层的 JSON 结构化剥离过程；前端界面同步生成包含图片描述、五维雷达图及改进建议的最终评估面板，直观呈现核心价值。
